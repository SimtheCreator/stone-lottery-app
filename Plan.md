# Stone Lottery App Plan

เป้าหมายระยะยาวคือเปลี่ยนจากเว็บจองเลขให้เป็นกิจกรรม engagement ที่มีพิธีกรรม สนุก ลุ้น แชร์ได้ และมี finale ปลายซีซั่นให้คนกลับมาดูคำทำนายของตัวเอง

## Engagement Principles

- Suspense first: ให้คนรู้สึกว่ากำลังทำพิธี ไม่ใช่กด form ธรรมดา
- Ritual feedback: ทุก action สำคัญต้องมีจังหวะตอบสนอง เช่น crack, seal, engraving, log
- Social proof แบบพอดี: ให้เห็นว่าคนอื่นกำลังร่วมกิจกรรม แต่ไม่ทำให้รู้สึกแข่งขันแรงเกินไป
- Shareable result: หลังเลือกเลขควรมีผลลัพธ์ที่อยากแคปหรือแชร์
- Performance before decoration: ลูกเล่นต้องไม่ทำให้การแย่งกดเสียจังหวะ
- No feature creep: เพิ่มเฉพาะลูกเล่นที่ทำให้ลุ้น สนุก หรือกลับมาดูซ้ำ

## Phase 1: Production Readiness

- ทำให้ GitHub push และ Vercel deploy flow เสถียร
- เช็ก production env vars และ Firebase Admin config ให้ครบ
- Smoke test หลายคนพร้อมกันบน production URL
- ยืนยัน Admin Dashboard ต้องถามรหัสก่อนเข้า
- ยืนยัน reset งวดเก็บ archive และ purge test data ล้างข้อมูล test ได้จริง
- ทำ checklist ใช้งานจริงก่อนเปิดกิจกรรม

## Phase 2: Premium UI/UX และ Performance

- ปรับศิลาให้ดูเป็นหินจริงขึ้นอีกโดยไม่เพิ่ม GPU cost มากเกินไป
- ทำ hover state ของศิลาที่ยังว่างให้เรืองแบบ neon magic ชัดขึ้น
- ทำศิลาที่ถูกผนึกแล้วให้แยกจากศิลาว่างชัดกว่าเดิม และรอยสลักชื่ออ่านง่าย
- ปรับ crack animation เป็น 3 จังหวะ:
  - hairline crack
  - shell split
  - final seal + name engraving
- ทำเสียง crack ให้ sync กับ animation โดยแยกชั้นเสียงเบาๆ แทนเสียงหนัก
- ลด animation ที่รันตลอดเวลา และใช้ transform/opacity เป็นหลัก
- รองรับ `prefers-reduced-motion`
- เช็ก mobile layout ก่อน deploy ทุกครั้ง

## Phase 3: Season Intelligence และ Personal Prediction

- ใช้ข้อมูลจาก `round_archives.participants` รวมกับกระดานปัจจุบัน
- สร้าง profile รายจอมเวทจากรหัสพนักงานเดิม
- วิเคราะห์ pattern ขั้นต่ำ:
  - เลขที่เคยเลือกทั้งหมด
  - ชอบเลขคู่หรือเลขคี่
  - ชอบเลขสูงหรือเลขต่ำ
  - ชอบเลขท้ายเดิมหรือไม่
  - เลือกเลขซ้ำหรือเปลี่ยนทุกครั้ง
  - เลือกเลขช่วงแคบหรือกระจายหลายช่วง
  - ความเร็วในการเลือก ถ้า timestamp เพียงพอ
- สร้าง archetype แบบสนุก เช่น:
  - นักย้ำชะตา
  - นักสำรวจวงเวท
  - ผู้คุมสมดุล
  - นักล่าปลายกระดาน
  - ผู้รักษาฐานพลัง
  - จอมเวทญาณไว
- ใช้ภาษาคำทำนายแบบ entertainment ไม่ใช่การตัดสินนิสัยจริงหรือ HR assessment

## Phase 4: Season Finale Experience

- เพิ่มหน้า `คำทำนายปลายซีซั่น`
- ให้ผู้ใช้กรอกรหัสพนักงานเพื่อดูคำทำนายของตัวเอง
- แสดงการ์ดสรุป:
  - ชื่อ/รหัสจอมเวท
  - archetype
  - เลขที่เคยเลือก
  - pattern เด่น
  - lucky number หรือเลขประจำซีซั่น
  - คำทำนายสั้นที่อ่านแล้วสนุก
- ทำ share card สำหรับ save/share เป็นรูปได้
- เพิ่ม leaderboard แบบเบาๆ เช่น ผู้เลือกเร็วสุด ผู้เลือกเลขหลากหลายสุด แต่ต้องไม่ทำให้คนรู้สึกถูกจัดอันดับ performance
- ทำ finale recap สำหรับ Admin เช่นจำนวนผู้ร่วมกิจกรรมทั้งหมด เลขยอดนิยม ช่วงเลขที่ถูกเลือกมากที่สุด

## Next UI/UX Ideas

### Board Experience

- เพิ่มโหมดพิธีเปิดกระดานตอนเข้า board ให้แสงวิ่งผ่านศิลา 100 แผ่นแบบเบาๆ
- เพิ่ม status band ว่าเลขถูกผนึกไปแล้วกี่เปอร์เซ็นต์ เป็นวงเวทเติมพลัง
- เพิ่มกลุ่มเลขที่ยังไร้ผู้ครอบครอง เพื่อช่วยให้คนเลือกต่อได้เร็ว
- เพิ่ม small pulse เฉพาะศิลาที่ยังว่าง ไม่ pulse ทุกแผ่นพร้อมกัน

### Selection Ritual

- หลังยืนยัน ให้มี sequence ชัดเจน: ตรวจสิทธิ์ -> crack -> seal -> engrave
- ระหว่างกำลังบันทึกให้ขึ้น microcopy เช่น `วงเวทกำลังตรวจสิทธิ์`
- เมื่อสำเร็จให้ขึ้น `ศิลายอมรับจารึกแล้ว`
- เมื่อไม่สำเร็จให้ขึ้น `ผนึกถูกครอบครองแล้ว` และแนะนำเลือกเลขอื่น

### Social Engagement

- พงศาวดารสดแสดงเฉพาะ 5-8 รายการล่าสุด
- เพิ่ม `จอมเวทล่าสุด` หรือ `ผู้ปลุกศิลาคนล่าสุด`
- เพิ่ม `เลขฮอต` จาก pattern ที่คนชอบเลือกใน archive
- เพิ่ม share card หลังเลือกเลข โดยไม่ต้อง login เพิ่ม

### Season-End Prediction

- หน้า prediction ต้องใช้รหัสพนักงานเดียวกับตอนเล่น
- ถ้าไม่มีข้อมูล ให้ตอบเป็นธีมเวทมนตร์ว่า `ยังไม่มีร่องรอยในศิลา`
- ถ้ามีข้อมูลน้อย ให้ทำนายแบบเบาๆ และบอกว่ายิ่งเล่นหลายงวดยิ่งแม่น
- ถ้ามีข้อมูลหลายงวด ให้สรุป pattern เป็น archetype และคำทำนายที่แชร์ได้

## Acceptance Criteria รอบ UI/UX ถัดไป

- Board ยังแสดง 100 แผ่นครบ
- Interaction หลักไม่ช้ากว่าเดิมจนรู้สึกได้
- จองเลขสำเร็จต้องมี feedback ภายในทันที
- Notice ต้องชัดและอยู่ในธีม
- Mobile scroll และปุ่มออกจากลานพิธีใช้งานได้
- Admin Dashboard ไม่พังจาก style ใหม่
